# AGENTS.md

## 项目定位

本项目是一个以 Flask 为核心的交易研究、策略回测与投资组合管理平台。
当前覆盖的主要能力包括：

- A 股 ETF 期权 Delta / 波动率回测
- 香农网格评分与回测
- A 股 / 美股 / 港股数据管理与波动率分析
- 国内 / 海外投资组合管理
- 财报期权策略分析
- 威科夫分析
- 大 V 跟投管理

## 关键入口

- 应用入口：`app.py`
- 路由层：`routes/`
- 策略层：`strategies/`
- 网格相关：`grid/`
- 服务层：`services/`
- 数据访问层：`db/`
- 页面模板：`templates/`
- 前端脚本：`static/js/`
- 工具脚本：`tools/`
- 手工验证脚本：`scripts/manual_tests/`

## 数据库约定

正式使用的 SQLite 数据库统一放在 `db/` 目录下。

主要数据库：

- `db/market_data.db`
- `db/market_data_min.db`
- `db/backtest_schemes.db`
- `db/portfolio.db`
- `db/us_stock.db`
- `db/hpc.db`

约束：

- 不要在项目根目录新增数据库副本
- 不要默认使用根目录下的 `*.db` 文件
- 新增数据库配置时，统一放到 `db/config.py`
- 除非用户明确要求，不要修改真实业务数据库内容

## 目录约定

### 根目录

根目录应尽量保持干净，只保留：

- 应用入口文件
- 项目说明文档
- 依赖与启动文件
- 必要的顶层模块

不要把以下内容放回根目录：

- 临时测试脚本
- 调试脚本
- 手工验证脚本
- 历史数据库副本
- 临时导入导出文件

### 手工验证脚本

所有一次性验证、数据源探测、调试脚本统一放在：

- `scripts/manual_tests/`

这些脚本不是正式自动化测试，不应视为 `pytest` 测试套件的一部分。

### 前后端结构

- 页面模板放在 `templates/`
- 页面级交互脚本放在 `static/js/`
- 新增页面时，优先遵循“模板 + 页面级 JS + 对应 route”的现有模式

## 修改约定

### 改后端功能时

优先检查以下位置：

- HTTP 接口：`routes/`
- 业务逻辑：`services/`、`strategies/`、`grid/`
- 数据访问：`db/`
- 通用工具：`utils/`

### 改前端页面时

优先检查以下位置：

- 页面模板：`templates/`
- 页面脚本：`static/js/`
- 页面入口路由：`routes/`

### 改数据库相关逻辑时

必须先确认：

- 当前代码实际使用的是哪个数据库文件
- 路径是否来自 `db/config.py`
- 是否会影响现有真实数据

## 文档约定

- 项目总览维护在 `README.md`
- 设计说明和业务文档维护在 `docs/`
- 如果调整了目录结构或模块职责，应同步更新 `README.md`

## Git 与清理约定

- `.idea/` 不纳入版本控制
- `.DS_Store` 不纳入版本控制
- 不提交本地缓存、临时文件或调试产物
- 对历史遗留文件做清理时，先确认是否仍被代码引用

## 给代理的建议

在修改本项目之前，优先做这几件事：

1. 先查真实入口和调用链，不要只看 README
2. 先确认数据库和文件路径是不是正式路径
3. 区分“正式模块”和“手工脚本”
4. 避免在根目录新增临时文件
5. 涉及结构调整时，同步更新文档
